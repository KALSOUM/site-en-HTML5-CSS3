les nouveaux balises d'html5



BALISES DESCRIPTIONS
<article> Définit l'emplacement d'un article. Il est generalement composé d'un titre et d'un paragraphe.
<section> définition une section représentant une thématique. un<article> peut etre composée de plusieurs thématiques (section), l'inverse également.
<aside> Permet de définir une section devant etre placer à cote de votre contenu principal. idéal pour placer les sidebars ou les regies publicitaires.
<figure> Est une conteneur fonctionant avec la balise <figcaption>, figure sert à afficher une image ou une vidéo suivie d'une légende (<figcaption>).
<footer> définit l'emplacement du pied de page de votre page.
<header> définit l'en-tete de votre page.
<hgroup> une balise qui permet de grouper plusieurs titres. <h1> <h6>
nav Représente une section contenant des liens. Elle est fréquemment utilisée dans le header pour définir le menu principal ou dans le footer.
Les balises Médias
<video> permet d'afficher un lecteur vidéo en HTML5, il faudra indiquer la source de votre vidéo l'attribut src
<audio> meme chose pour la balise précédente mais dans le cas d'un fichier audio uniquement.
<picture> la balise picture permet d'utiliser différentes <source> destinées à un élément <img> le navigateur choisira l'image la plus pertinente.
<track> Permet la spécification de pistes texte (piste sous-titres par exemple) pour un élément vidéo ou audio.Bien à connaître mais l'implémentation dans les navigateurs n'est pas encore universelle pour cette balise (remarque août 2014).
<source> Montre aux gens derrière l'idée et décrire brièvement leur rôle.
<embed> la balise embed permet d'utiliser un conteneur pour afficher les ressources externes, elle a été utilisée notamment pour afficher du flash, du svr, ...
<mark> Permet de souligner un texte.
<meter> Représente une jaune comprise entre deux valeurs fixe.
<progress> Similaire à meter mais représente un pourcentage de progression.
<time> Permet de représenter d'une maniére sémantique une date, elle accepte comme attribut datetime
<progress> <wbr> Permet d'indiquer à votre navigateur de réaliser si besoin un retour à la ligne à partir de cette balise.
Les balises essentilles: TEXTES
<h1>, <h2> ...<h6> Ces balises permettent de marquer du texte en tant que titre. Les titres sont affichés avec un style différent du texte ordinaire - typiquement plus grand. Le contenu d'un élément h1 est typiquement affiché plus grand que celui d'un élément h2, qui est plus grand que h3, et cetera, avec h6 typiquement le plus petit.
<p> La balise p permet d'organiser les textes en paragraphe (effet de double retour à la ligne automatique entre les paragraphes à l'affichage).
<br> <hr> La balise br permet de mettre un simple retour à la ligne dans un texte (à l'intérieure d'un paragraphe par exemple). La balise hr permet de dessiner une ligne horizontale sur toute la largeur de la page. Ce sont des éléments vides (pas de contenu, ni de balise de fermeture).
Les balises essentilles: LIENS
<a> C'est ce qui met le "hyper" dans notre hypertexte. La balise est la balise pour créer un hyperlien. Sans hyperlien le World Wide Web n'existerait pas.
Les balises essentilles: IMAGES
<img> Balise pour afficher une image. C'est un élément vide (pas de contenu ni de balise de fermeture).L'adresse de l'image est spécifié dans l'attribut src. Le navigateur utilise cette adresse pour trouver le fichier. Les chemins (tout comme pour les liens) peuvent être relatifs ou absolus. On applique les même règles.
L'attribut alt L'attribut alt est obligatoire pour une image (pour la balise img) Selon le navigateur le texte alt prend la place de l'image quand elle n'est pas disponible (par exemple si le site sur lequel l'image est stocké est en panne)
Les balises essentilles: TABLES et LISTES
<table>, <th>, <tr>, et <td> La balise table est la balise à la racine. La balise th dénote un titre (h=heading), tr dénote une ligne (r=row), td dénote une cellule (d=data)
<ul>, <ol>, <li> alises pour créer: des listes ordonnées (ol = ordered list), et des listes non-ordonnées (ul = unordered list). Dans les deux cas, chaque élément de la liste est encapsulé par la balise li (li=list item).
Caractéres spéciaux et Entités HTML
&nbsp; Permet de mettre littéralement un espace supplémentaire.
<!DOCTYPE> Utilité du doctype: Tout simplement, le navigateur utilise la déclaration doctype pour savoir à quelle version de HTML s'attendre. Si on déclare du HTML 4.0 strict mais on utilise des balises nouvelles de HTML5, il peut y avoir des problèmes à l'affichage. Ceci est important à savoir quand on pratique du copier-coller du code source sur internet. Il faut faire attention au doctype qu'on colle.
Doctype et validation:Le W3C HTML Validator exploite également la déclaration doctype. La validation d'un document dont la version de HTML utilisée et la version déclarée ne sont pas cohérentes échoue .
